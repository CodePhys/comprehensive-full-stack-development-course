Git & GitHub

Istruzioni principali

I comandi principali utilizzati in Git e GitHub sono fondamentali per gestire il versionamento del codice e collaborare con altri sviluppatori. Ecco una breve descrizione dei comandi più comuni:

  1. git init: Inizializza un nuovo repository Git nella directory corrente. Questo comando crea una nuova cartella nascosta chiamata ".git" che conterrà tutti i dati di versionamento.
  2. git clone repository_url: Clona un repository Git esistente sul tuo computer. Questo comando crea una copia locale del repository remoto, consentendoti di lavorare sul codice in modo isolato.
  3. git add file_path: Aggiunge un file specifico alla staging area. La staging area è un'area temporanea in cui vengono preparati i file per il commit.
  4. git commit -m "commit_message": Esegue un commit dei file presenti nella staging area. Un commit è un'operazione che registra le modifiche apportate ai file nel repository Git.
  5. git push origin branch_name: Carica i commit locali sul repository remoto specificato. Questo comando sincronizza il repository locale con il repository remoto.
  6. git pull origin branch_name: Aggiorna il repository locale con le ultime modifiche dal repository remoto. Questo comando è utile per ottenere le ultime modifiche effettuate dagli altri sviluppatori.
